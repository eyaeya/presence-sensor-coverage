# 人在传感器安装定位可视化仿真工具 — 设计文档

日期：2026-05-19
形态：单一 `index.html` 文件（SVG + 原生 JS，无构建、无依赖）

---

## 1. 目标

为家装场景提供一个实时、可交互的顶视图工具，仿真"人在传感器（毫米波雷达）"在不同安装方式、安装位置、安装角度、FOV 与探测距离下，对房间的覆盖情况。所有条件都有默认值，计算开销极低，**所有改动实时重算并重绘**。

## 2. 范围

**做：** 矩形封闭空间；吸顶 / 侧装 / 墙角三种安装；FOV 与双探测距离可调；4 个被检测高度的洋葱分层覆盖 + 两条探测边界虚线；安装定位信息栏；鼠标距墙读数。

**不做（YAGNI）：** 非矩形/异型空间；自定义被检测高度档位（固定站1200/坐750/躺600/地0）；多传感器；导出/持久化；3D 视图。

最大探测距离沿**传感器正面垂直线（波束中轴）**度量（轴向距离，非径向）。
侧装 ψ 上限 0–±90° 为本设计假设，审阅时可改。

## 4. 坐标系

- 顶视：`x∈[0,W]` 向右，`y∈[0,D]` 向上。
- 墙映射：左 `x=0` / 右 `x=W` / 下 `y=0` / 上 `y=D`。
- 第三维 `z` = 离地高度，`z∈[0, 安装高度]`。传感器 3D 位置 `S=(sx,sy,sz)`。

## 5. 波束模型

波束 = **椭圆锥**：中轴单位向量 `d`，局部正交基 `(u,v)`，`u` 承载水平 FOV（半角 αH=HFOV/2），`v` 承载垂直 FOV（半角 αV=VFOV/2）。点 `P` 在锥内当且仅当（`r=P−S`，`t=r·d>0`）：

```
(r·u / (t·tan αH))² + (r·v / (t·tan αV))² ≤ 1
```

距离截断：带探测距离的边界，附加 `t ≤ R`（R 为对应的存在/运动距离），即垂直于中轴、轴向距离 R 处的平面截断。

### 5.1 各安装方式的中轴 d 与局部基

- **吸顶**：`d=(0,0,−1)`（正下）。`u,v` 在水平面内，由水平角 φ 绕竖直轴旋转。
- **侧装**：所附墙的内法线（水平、指向室内）→ 偏航 ±ψ → 下倾 θ。`u` = 水平且垂直于 d 水平投影（位于地面方向，承载 H-FOV）；`v = d×u`（承载 V-FOV / 下倾展开）。
- **墙角**：夹角平分线 45°（固定，指向室内）→ 下倾 θ。局部基同侧装。

### 5.2 顶视足迹形态（已与用户确认）

- **吸顶**：以传感器正下方 `(sx,sy)` 为圆心的**同心、共朝向椭圆**；φ 转椭圆朝向；h 越低圈越大 → 真·洋葱圈。FOV 大时基本铺满全屋（如实）。
- **侧装**：从墙上传感器点**射入室内的瓣状区**；非同心，共墙上根部；**随距离横向展开变宽**（大 H-FOV 扇面，不会伸远却收窄）；h 越低瓣越大伸越远；ψ 转动整个瓣。
- **墙角**：从墙角**沿 45° 平分线射入室内的瓣状区**；**始终指向室内，绝不指向室外**；嵌套共墙角根部。

## 6. 核心可视化语义

对某被检测高度 h：波束在水平面 `z=h` 的截面，竖直投影到地面 = 该高度的覆盖足迹。

绘制内容：

1. **4 层洋葱填充**（站1200/坐750/躺600/地0）：纯 FOV 几何，**只裁到房间墙**（不卡探测距离）。绘制顺序：地0（最大）→躺600→坐750→站1200（最小），半透明叠加。
2. **蓝色虚线（存在检测边界）**：足迹 @ h=750mm ∩ 存在最大距离，再裁到房间。
3. **橙色虚线（运动检测边界）**：足迹 @ h=0mm ∩ 运动最大距离，再裁到房间。

**强制硬规则：** 所有填充与虚线边界一律硬裁剪到房间矩形内，房间墙外不渲染任何像素。几何层用 Sutherland–Hodgman 多边形裁剪，渲染层叠加 SVG `clipPath`=房间矩形，双保险。

## 7. 架构

单文件内 5 个低耦合模块（独立对象/区块）：

| 模块 | 职责 | 依赖 |
|---|---|---|
| `state` | 唯一数据源 | 无 |
| `geometry` | 纯函数：参数 → 地面多边形 | 仅数学，无 DOM |
| `render` | 读 state+geometry → 画 SVG | state, geometry |
| `interaction` | 拖拽/点击/滑杆/输入 → 改 state → 触发重算 | state |
| `infoPanel` | 格式化安装定位 + hover 距离 | state |

数据流：用户操作 → `interaction` 改 `state` → `render`（内部调 `geometry`）→ 更新 SVG 与 `infoPanel`。全程同步实时。

### 7.1 状态对象

```js
state = {
  room:   { W:4000, D:3000 },
  mount:  'ceiling' | 'side' | 'corner',
  sensor: { x, y },        // 顶视位置；侧装锁墙、墙角锁角
  wall:   'left'|'right'|'top'|'bottom',  // 侧装所附墙
  corner: 'bl'|'br'|'tl'|'tr',            // 墙角所在角
  height, tilt, hAngle,    // 安装高度 / 下倾角 / 水平角
  hFov:160, vFov:90,
  rangePresence:3000, rangeMotion:5000
}
```

切换 `mount` → `sensor/height/hAngle` 重置为该方式默认值，无关控件禁用。

## 8. 几何引擎算法

给定（安装方式、S、d、u、v、αH、αV、检测高度 h、可选 R）求地面多边形：

1. 绕锥面采样 β∈[0,2π) 约 240 点，边界射线方向 `g(β)=normalize(tanαH·cosβ·u + tanαV·sinβ·v + d)`。
2. 每条射线与平面 `z=h` 求交：`t=(h−S_z)/g_z`；若带 R，则 `t←min(t, 轴向R对应t)`。
3. `g_z≥0`（射线朝上/水平，不与下方平面前向相交）→ 该方向延伸到远界（10× 房间对角线），靠房间裁剪收口。
4. 收集点成多边形 → Sutherland–Hodgman 裁剪到房间矩形 `[0,W]×[0,D]`。
5. 采样法天然覆盖椭圆/抛物/双曲各情形，避免脆弱的二次曲线分类代数。

## 9. 渲染（SVG 分层）

单 SVG，`viewBox` 映射房间 mm 并保持 W:D 比例，四周留边以便抓取墙上手柄。统一 `mm→px` 变换为唯一真源。z 序：

1. 房间矩形 + 墙 + 尺寸标注
2. 4 层洋葱填充（`clipPath`=房间矩形）
3. 橙虚线（运动）→ 蓝虚线（存在）
4. 传感器手柄（可拖圆点）+ 朝向短箭头
5. 鼠标位置由 `infoPanel` 文本呈现

配色：站=紫`#9b8cff`、坐=蓝`#5fb0ff`、躺=青`#5fe0c0`、地=黄`#f5d05a`；存在=蓝虚线`#4f9bff`、运动=橙虚线`#f0913a`。深色技术风，图例常驻，传感器=白点+细朝向线。布局：主区 flex 放大 / 右栏 ~280px / 下栏定高换行。

## 10. 交互

- **吸顶**：屋内任意拖传感器，钳制到 `[0,W]×[0,D]`。
- **侧装**：传感器锁墙；点任意墙 → 吸附到点击处并记录所附墙；拖动 → 沿当前墙滑动。所附墙内法线 = ψ 的 0°。
- **墙角**：点击就近墙角 → 跳转该角；不可拖；45° 固定。
- 工具栏控件输入框+滑杆双向同步，控件随安装方式启用/禁用（吸顶禁用下倾角；墙角水平角显示"45° 固定"禁用）。

## 11. 信息栏（右）

1. **安装定位**
   - 吸顶：安装方式、距上/下/左/右墙、安装高度。
   - 侧装：安装方式、距所附墙两端距离、安装高度、下倾角度。
   - 墙角：安装方式、（不显示安装距离）、安装高度、下倾角度。
2. **当前参数**：H/V FOV、存在/运动距离速览（用户确认保留）。
3. **鼠标位置**：hover 屋内时实时显示距上/下/左/右墙(mm)，移出清空。

单位一律 mm。

## 12. 边界情况

- 输入钳制到各自 min/max，拒绝 NaN，回退上次有效值。
- 房间太小 / FOV 过大 → 足迹全裁进屋（铺满），属正常非报错。
- 波束上沿朝上（θ−αV<0）→ 不相交射线延伸至远界由房间裁剪收口；带 R 的虚线再用最大距离平面截断；不崩溃。
- 某层检测高度 `h ≥ 安装高度` → 该层不绘制（物理正确，如墙角高度 1000 时"站1200"层为空）。
- 侧装在不同墙：按墙计算内法线；墙角：按角计算 45° 平分线。

## 13. 测试

- `geometry` 为纯函数，加 `?test` 模式跑断言并在 console 输出通过/失败：
  1. 吸顶 φ=0：足迹为以 `(sx,sy)` 为心、半轴 `(H−h)·tanαH`、`(H−h)·tanαV` 的椭圆（验中心与半轴）。
  2. 距离截断：吸顶 `H−h>R` 时存在边界为空；`<R` 时等于该高度 FOV 椭圆。
  3. 裁剪：足迹大于房间时多边形等于房间矩形。
  4. 侧装 ψ=0 在左墙：足迹关于 `y=sy` 对称。
- UI 正确性以**真实浏览器实测**为准：黄金路径（三种安装默认态）+ 边界（极值 FOV/距离、拖到墙角、切换安装、房间极小）逐一验证，并确认无足迹溢出墙外。
